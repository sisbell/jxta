Releases and Milestones

The JXTA Java SE/SE (JXSE) release team has committed to a regular release schedule. For several years we produced releases on a quarterly schedule. The project now schedules thrice yearly (three times a year) releases. A fixed release schedule provides well tested, stable releases containing updates and fixes on a regular basis. Releases will contain a balance of fixes and features which will depend entirely on their readiness for inclusion.

JXSE releases are named after exotic and exciting cuisine. (You are welcome to suggest release names). Within each release cycle there are a number of important milestones:

  • New Code Freeze : This is the last date at which new features and new code will be accepted into the release. Following this date all development on the release will involve fixing regressions and high priority issues. For quarterly releases the "Code Freeze" date is normally five weeks before the planned release date. For the thrice yearly release schedule this will normally be seven weeks before the planned release date.

    At "New Code Freeze", all non-DEFECT (FEATURE/ENHANCEMENT) issues which are not already marked for review will be deferred for future releases.


  • Beta Release : This is the date at which the broader community of JXTA users will be asked to evaluate and test the upcoming release. Community members may, of course, test and make comments at any time. Beta releases are intended primarily to uncover any hidden regressions and problems not previously encountered by the release team. Dates shown below in parentheses are the planned or expected dates The beta release occurs a minimum of 10 days before the planned release date (usually more).

    At "Beta Release" all uncommitted non-DEFECT issues (FEATURE/ENHANCEMENT) will be deferred to future releases. All P3 and lower priority DEFECT issues will also be deferred for future releases.


  • Release : This is the date at which the release is expected to be finalized. These releases will have a very high level of testing and should be suitable for all uses of JXTA. Dates shown below in parentheses are the planned or expected dates.

    At "Release" there may be no active P1 or P2 DEFECT issues and no other issues may be assigned to the release.
